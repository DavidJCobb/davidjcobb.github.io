Suppose you have an expression of the form (a + b + c + d + e + f){{lu_math_sq}}, and you need to expand the square in order to solve some larger equation. How do you actually do that? More generally, is there an approach we can use to expand any group of added-together terms with any exponent?

Yes. It's called the multinomial theorem.

Let's get started

The multinomial theorem can be used to expand any expression of the form (x1 + x2 + ... + xm){% include lu-math/exponent.html exponent=var_Nsca %}. Let's take a look at the equation:

(x1 + x2 + ... + xm){% include lu-math/exponent.html exponent=var_Nsca %} = Σ for all k1 + k2 + ... + km = {{var_Nsca}}; k1,k2,...,km ≥ 0 do: ({%- include lu-math/binomial-coefficient.html top=var_Nsca bottom="k1,k2,...,km" -%} × Π from t = 1 through m do: (xt{% include lu-math/exponent.html exponent="kt" %}) )

Oh. Uh...

Well, to start with, those values below the summation symbol (Σ) represent conditions. We have {{var_Msca_inl}} variables named ksomething, named k1 through km, and we have two conditions that apply to the lot of them. We are going to take every possible combination of values that meets these conditions, feed those values into the summation "body," and then compute the results and sum them all.

The summation "body," meanwhile, has us multiply a collection of values — that's the capital-pi (Π) symbol; it's the multiplication version of Σ — and then multiply that by something else. The result is then added as part of the summation loop.

The first condition on our summation loop is that the values must all add up to {{var_Nsca_inl}}, and the second condition is that they must all be greater than or equal to zero. This in turn implies that no one single value can be greater than {{var_Nsca_inl}}, as otherwise we'd need at least one of the others to be negative in order for this to work.

See the large parentheses, with the two rows and no symbol dividing them? Hear that one bit with the word "choose?" That's the notation for binomial coefficients. If we call the top number {{var_Nsca_inl}} and the bottom number {{var_Ksca_inl}}, then given the expression (1 + {{var_Xsca}}){% include lu-math/exponent.html exponent=var_Ksca %}, we can compute the coefficient of {{var_Xsca}}{% include lu-math/exponent.html exponent=var_Ksca %} as follows:

{{var_Nsca}}! ÷ ({{var_Ksca}}!({{var_Nsca}} - {{var_Ksca}})!)

Apparently, that generalizes to larger "multinomials;" if your multinomial has {{var_Rsca_inl}} terms in it, then you'd do:

{%- include lu-math/binomial-coefficient.html top=var_Nsca bottom="k1,k2,...,kr" -%} = {{var_Nsca}}! ÷ (k1!k2!...kr!)

Given a multinomial with {{var_Rsca_inl}} terms, the variable ki is defined below — and as a reminder, this is summation, not a single variable:

Σ from i = 1 through r do: (ki) = {{var_Nsca}}

This is all getting pretty out-there, so let's come back to Earth by actually using this.

An example

We'll go with the example I led in with: ({{var_Asca}} + {{var_Bsca}} + {{var_Csca}} + {{var_Dsca}} + {{var_Esca}} + {{var_Fsca}}){{lu_math_sq}}. We have a multinomial that we want to square, so {{var_Nsca_inl}} is 2. Our multinomial has six terms, so {{var_Msca_inl}} is 6.

(x1 + x2 + x3 + x4 + x5 + x6){{ lu_math_sq }} = Σ for all k1 + k2 + ... + k6 = 2; k1,k2,...,k6 ≥ 0 do: ({%- include lu-math/binomial-coefficient.html top="2" bottom="k1,k2,...,k6" -%} × Π from {{var_Tsca}} = 1 through 6 do: (xt{% include lu-math/exponent.html exponent="kt" %}) ) (x1 + x2 + x3 + x4 + x5 + x6){{ lu_math_sq }} = Σ for all k1 + k2 + ... + k6 = 2; k1,k2,...,k6 ≥ 0 do: (2! ÷ (k1!k2!...k6!) × Π from {{var_Tsca}} = 1 through 6 do: (xt{% include lu-math/exponent.html exponent="kt" %}) )

This equation is a nested loop. The outermost loop has six iterator variables, and the innermost loop just goes over those. It's a bit like this: 

int[...][6] all_possible_k_arrays = ...;

int[6] k;

// summation (big weird "E" symbol):
sum = 0;
for each k[1], k[2], k[3], k[4], k[5], k[6] matching conditions, do:

   // capital-pi thing:
   product = 1;
   for t = 1 to k.length do
      product *= ...;
   end
   
   // multiply by factorial fraction:
   product *= ...;
   
   sum += product;
end

We need to start at the innermost loop and work our way outward to understand this, but the innermost loop uses the six "loop index" variables, ki, that belong to our outer loop. That means that we need to figure out what those variables are.

Dealing with the inner loop

That's a lot of combinations up there, but let's actually use them and see how that shakes out. The ki variables are used in two ways:

  • For each ki in any valid set of values, we compute xi{% include lu-math/exponent.html exponent="ki" %}. We then multply all of these together; this is the inner loop, called a sequence of products and indicated with the capital pi (Π).
  • After the inner loop is done, we compute the multinomial coefficient (the fraction with the factorials in it), and multiply that with the sequence of products.

Conclusion

This overall process can be followed for a multinomial of any size {{var_Msca_inl}}, being raised to any exponent {{var_Nsca_inl}}. You want to create a table like the ones above, listing out every combination of {{var_Msca_inl}} many numbers (each between 0 and {{var_Nsca_inl}}) that add up to {{var_Nsca_inl}}. You can then mimic the process above, and get your result.
